# Підхід та ключові рекомендації

Цей документ фіксує узгоджений підхід, який використано у проєкті: як безпечно відкривати Word-документи, як нормалізувати контент і як переносити текст/таблиці/зображення у новий документ зі стилями шаблону.

## 1) Правильне відкриття Word-документів (COM)

Мета: уникнути діалогів, попереджень, Protected View, блокувань.

Рекомендовані параметри:
- ConfirmConversions=False
- ReadOnly=True або False
- AddToRecentFiles=False
- OpenAndRepair=True при збоях
- Visible=False

Приклад:
```
Documents.Open(
    FileName=path,
    ConfirmConversions=False,
    ReadOnly=True,
    AddToRecentFiles=False,
    OpenAndRepair=False,
    Visible=False
)
```

Закриття:
```
document.Close(False)
word.Quit()
pythoncom.CoUninitialize()
```

## 2) Чому не можна copy/paste між документами

Word переносить приховані структури:
- стилі
- numbering.xml
- шаблони списків
- приховані таблиці стилів

Це викликає:
- автонумерацію
- паразитні стилі
- “залипання” форматування

## 3) Правильна нормалізація (Parse → Normalize → Rebuild)

### Принцип
Не копіювати Word-об’єкти. Потрібно:
1) прочитати структуру документа
2) зібрати новий документ з чистого контенту

### Алгоритм
1. прочитати `word/document.xml`
2. зчитати списки з `word/numbering.xml`
3. для абзаців брати лише `paragraph.text`
4. списки перетворювати у текстові маркери (•, 1., a))
5. таблиці створювати заново без стилів
6. зображення витягувати з `word/media` і вставляти у новий документ

### Результат
- 0 перенесених стилів
- 0 автонумерацій
- чистий документ для подальшої верстки

## 4) Перенесення тексту, таблиць, зображень

### Текст
- створювати новий абзац у новому документі
- вставляти лише текст
- не копіювати Paragraph object

### Таблиці
- створювати таблицю через `add_table(rows, cols)`
- копіювати лише текст клітинок
- не переносити table styles

### Зображення
- витягувати binary з `word/media`
- вставляти через `add_picture`

## 5) Застосування стилів нового документа

Після нормалізації, при формуванні чернетки:
- використовується шаблон `Jurnal.dotx`
- стиль секції: Heading 1 (або custom section)
- стиль авторів: Heading 2
- стиль назви: Heading 3

## 6) Висновок

Підхід гарантує:
- стабільну обробку документів різної якості
- відсутність паразитних стилів
- контрольований вигляд журналу
- можливість подальшої автоматичної верстки
