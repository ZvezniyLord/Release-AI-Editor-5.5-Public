# EXECUTION_FLOW

## Основне ядро (`ядро-реліз-інтегр`) — покроково
1. **Старт**: `launcher.py` (GUI/CLI).
2. **Дашборд**: `dashboard_core.prepare_run()` → `matches.json`, `manifest.json`, `dashboard_perevirky.xlsx`.
3. **Переміщення статей**: `core.relocate_articles` → `run_dir/Статті`.
4. **Базова нормалізація файлів**: `core.normalize_folder` → `run_dir/Статті_норм` (parse → normalize → rebuild).
5. **Маркерні та caption-блоки всередині статті**: `block_pipeline` виконує `markers_block` → `caption_block`.
6. **Шапка (header/shapka)**: `core.normalize_main` запускає `header_block`, який уніфікує авторів, афіліації, UDC/DOI та службові рядки.
7. **Порядок front-matter**: `order_block` збирає цілісний блок `UDC/DOI → шапка → назва`, прибирає випадкові порожні абзаци в межах front-matter і тільки після цього додає контрольовані відступи.
8. **Вирівнювання назв**: `title_style_apply.normalize_titles_docx` застосовує стиль `Назва1` уже після перестановки блоків.
9. **Оновлення dashboard-артефактів**: `update_dashboard_from_run` синхронізує `cleaned_path`.
10. **Чернетка**: `core.build_draft` → `draft_core.word_draft_builder`.
11. **Автори у чернетці**: `core.normalize_authors` (стиль `AUTOR`).
12. **Назви у чернетці**: `core.normalize_titles` (стиль `Назва1`).
13. **Зміст**: `core.toc_main` → `toc_core` + `Table.docx`; для інтегрованої кнопки `5. Додати таблицю змісту` використовується шаблон, вказаний у `launcher.py`, а заголовок блоку вільних слухачів генерується окремим рядком і має стиль секції (`Tab_SEC`).
14. **Додаткові інструменти**: `normalize_annotation_markers` / `normalize_reference_markers`.

### Примітки щодо вхідних форматів і секцій
- Сканер заявок приймає `docx`, `doc`, `dot`, `rtf`, `odt`; формати не-`docx` конвертуються у `docx` через Word COM (за `convert_mode=auto/com`).
- Мапінг секцій з Excel виконується за нормалізованою назвою секції (з урахуванням префіксів на кшталт `048.` та варіантів `і/та`); число без префікса `секція/section/block` не вважається номером секції.
- Для рисунків діє правило: один порожній абзац перед абзацом із графічним об'єктом і один порожній абзац між рисунком та абзацом-маркером `Рис. N`; якщо підпис продовжується маркованими рядками `• ...`, зайвий blank між маркером і продовженням не додається. Для таблиць стиль підпису застосовується до текстового абзацу після маркера `Таблиця N`.
- Для `References` нормалізація має розпізнавати також змішані варіанти маркера на кшталт латинської `C` у `Cписок` і не лишати хвостових порожніх абзаців у кінці reference-блоку.
