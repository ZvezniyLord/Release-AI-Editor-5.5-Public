# MODULE_DESCRIPTIONS

## Основне ядро (`ядро-реліз-інтегр`)
- **core pipeline**: `core/*` керує сценаріями relocate/normalize/draft/authors/titles/toc.
- **GUI**: `launcher.py` запускає пайплайн і передає параметри.
- **Документ‑скан**: `dashboard_core/*` читає Excel і сканує папку заявок.
- **Матчинг**: `dashboard_core/core/matcher.py` зіставляє статті з файлами.
- **Word COM**: `draft_core/word_draft_builder.py` вставляє статті у шаблон без втрати об'єктів.
- **Нормалізація**: `core/normalize_docx.py` перебудовує контент (parse → rebuild) і зберігає Office Math/OMath формули через плейсхолдери та подальше відновлення Word COM.
- **TOC/зміст**: `core/toc_main.py` + `toc_core/*`; інтегрована кнопка `5. Додати таблицю змісту` запускає саме цей шлях, а заголовок блоку `free_listener` формується з `manifest.json` / `sections_path` і стилізується як секція (`Tab_SEC`).
- **Шапка (authors/UDC)**: `shapka_core/shapka_main.py`.

## TOC‑ядро (`ядро GPT Super Табел`)
- **outline_parser.py** — аналіз структури документу.
- **table_builder.py** — формування рядків TOC.
- **main.py/gui.py** — CLI/GUI для додавання таблиці змісту.

## Legacy‑ядро (`ядро-реліз`)
- **launcher.py** — базовий автономний запуск (дашборд, relocate, чернетка).
- **core/** — старіший, але робочий набір скриптів.
## Single Article
- `ядро-реліз-інтегр/core/normalize_single_article.py` — нормалізація одного документа зі застосуванням введеної оператором назви статті, включно з маркерами анотації/ключових слів.
- `ядро-реліз-інтегр/core/normalize_caption_layout.py` — правила для підписів `Таблиця/Table` та `Рис./Figure`: для рисунків підтримує один порожній абзац перед самим рисунком, один між рисунком і абзацом-маркером `Рис. N` та не розриває марковане продовження caption-блоку; для таблиць стиль підпису застосовується до наступного текстового абзацу після маркера `Таблиця N`.
- `ядро-реліз-інтегр/core/block_handlers/order_block.py` — перестановка front-matter блоків (`UDC/DOI` → цілісна шапка → назва), очищення випадкових порожніх абзаців у першому блоці статті та підготовка стабільного кейсу для багаторядкових афіліацій.
