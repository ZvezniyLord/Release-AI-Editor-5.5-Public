# Бізнес-логіка нормалізації маркерів (короткий звіт)
## 1) Мета
Уніфікувати маркери (Анотація/Keywords/References) і привести їх до єдиного формату, без прив’язки до реалізації скриптів.
## 2) Загальні правила нормалізації
1. **Розпізнати маркер** на початку рядка (префіксна перевірка).
2. **Нормалізувати маркер**:
   - прибрати службову нумерацію на старті (1., 2), 3-);
   - прибрати зайві пробіли та пунктуацію з початку/кінця;
   - порівняння вести в lower-case.
3. Якщо після маркера йде **Enter**, то текст із наступного непорожнього абзацу **підтягується** в цей рядок.
4. **Єдиний вихідний формат**:
   - Маркер(пунктуація) + пробіл + текст.
5. **Маркер виділяється жирним**, а текст після нього — звичайним.
6. **Порожній абзац гарантується лише після Keywords.** Після анотації порожній абзац **не допускається**.
## 3) Правило пунктуації в маркері
- Якщо в replacement уже задано пунктуацію (: . ; ! ?) — вона зберігається.
- Якщо пунктуації нема — додається :.
- Після маркера завжди додається пробіл.
## 4) Підтягування тексту після Enter
Якщо маркер стоїть окремим рядком:
1. Береться найближчий змістовний абзац нижче.
2. Цей абзац очищається.
3. Текст вставляється після маркера в один рядок.
## 5) Списки маркерів (готові масиви)
### 5.1 Annotation / Abstract
**Маркерні варіанти:**
- "Анотація"
- "Annotation"
- "Анотація:"
- "Abstract:"
- "Анотація. / Annotation."
**Заміни (стандартні заголовки):**
- uk: "Анотація."
- en: "Abstract."
### 5.2 Keywords
**Маркерні варіанти:**
- "Ключові слова"
- "Keywords"
- "Key words:"
- "Ключові слова. / Keywords."
**Заміни (стандартні заголовки):**
- uk: "Ключові слова:"
- en: "Keywords:"
### 5.3 References
**Маркерні варіанти:**
- "Список використаних джерел"
- "Література"
- "References"
- "List of reference"
- "List of references"
- "Список використаних джерел:"
**Заміни (стандартні заголовки):**
- uk: "Список використаних джерел:"
- en: "References:"
### 5.4 Блокові маркери (межі секцій)
**UK:**
- "Вступ"
- "Мета"
- "Матеріали та методи"
- "Результати"
- "Висновки"
**EN:**
- "Introduction"
- "Aim"
- "Materials and methods"
- "Results"
- "Conclusions"
## 6) Додаткове правило для Keywords
Перше слово після маркера робиться з **великої літери**.
Також після блоку Keywords має бути **рівно один порожній абзац**.
---
# Висновок
Бізнес-логіка маркерів — це стабільний набір правил для:
- розпізнавання маркера,
- приведення пунктуації,
- підтягування тексту після Enter,
- уніфікації формату вихідного рядка.
Ці правила можна перенести у ваше ядро незалежно від реалізації.
