# AGENT_CONTEXT

## Короткий опис
Release AI Editor — набір модулів для складання наукового журналу з папки заявок та Excel‑реєстру учасників. Система сканує Word‑файли, зіставляє їх з назвами статей, збирає чернетку через Word COM і формує дашборд та технічні звіти.

## Основні ядра
- `ядро-реліз-інтегр` — головний автономний пайплайн (dashboard → relocate → normalize → draft → authors/titles → TOC).
- `ядро GPT Super Табел` — генерація таблиці змісту.
- `ядро-реліз` — legacy‑ядро (старіший шлях).

## Ключові маркери та ключові слова
- Маркери: UDC/УДК, Анотація/Abstract, Ключові слова/Keywords, References/Список джерел.
- Плейсхолдери: `<<CHART_PLACEHOLDER_N>>`, `<<IMG_PLACEHOLDER_N>>`.
- Стилі: `SECTION`, `AUTOR`, `Назва1`, `REFER`, `REF-TITLE`, `TABLETEXT`, `Tab_*`.
- Артефакти: `matches.json`, `manifest.json`, `summary.json`, `diagnostics.json`, `dashboard_perevirky.xlsx`, `draft_journal.docx`.

## Високорівневий execution flow
1. Excel → список статей.
2. Скан папки заявок → кандидати.
3. Матчинг → `matches.json`.
4. Чернетка через Word COM → `draft_journal.docx`.
5. Дашборд + JSON + технічні звіти.
6. (Опційно) Нормалізація, автори/назви, TOC.

## Операційний контекст для нових чатів
- Користувач часто звертається короткими фразами на кшталт `не перенісся Малюнок`, `тут зламався DOI`, `пуш`, `відміна змін`; це зазвичай означає конкретний збій у пайплайні на реальному файлі, а не абстрактне питання.
- Основна цінність програми: напівавтоматично готувати випуск журналу з реальних Word-документів без тихих втрат контенту, форматування або службових маркерів.
- Для збоїв перенесення графіки, формул, таблиць, OLE/chart-об'єктів і нестандартних drawings безпечніше лишити видимий плейсхолдер або інший явний сигнал оператору, ніж тихо прибрати проблемне місце.
- Якщо агент змінює поведінку fallback-механізмів, треба зберігати принцип `fail visible`: оператор має побачити, що об'єкт не перенесено автоматично.
- Типові робочі кейси: нормалізація окремої статті, пакетна нормалізація папки `Статті` -> `Статті_норм`, збірка чернетки журналу, відновлення загублених елементів Word, правка стилів/шапки/UDC/DOI, діагностика mojibake та маркерів.
- Якщо користувач дає шлях до конкретного `docx`, треба перевіряти і сам файл, і відповідний код пайплайну; можливий як одноразовий ремонт файлу, так і системна правка механізму.
