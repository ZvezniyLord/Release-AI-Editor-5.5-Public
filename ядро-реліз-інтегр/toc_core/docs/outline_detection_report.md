# Звіт: як виявляти рівні 1/2/3 у Word

## 1) Надійні сигнали рівня (у пріоритеті)
1. **Outline Level (Word COM)** — найточніше джерело і відповідає тому, що видно у Navigation Pane. Якщо OutlineLevel = 1..3 — це рівні 1..3. Усі стилі з outline level 1–9 відображаються у Navigation Pane. citeturn1search0turn1search4turn1search7turn1search11
2. **`w:outlineLvl` у `document.xml`** — прямий рівень абзацу (0‑based; 0=Level 1, 1=Level 2, 2=Level 3). Його треба читати з `w:pPr/w:outlineLvl`.
3. **`w:outlineLvl` у `styles.xml`** — якщо в абзаці немає outlineLvl, але стиль має `w:pPr/w:outlineLvl` (через `basedOn`), то цей рівень застосовується. citeturn1search1turn1search2
4. **Назва стилю (Heading 1/2/3, Заголовок 1/2/3)** — запасний евристичний варіант, бо стилі можуть бути перейменовані. citeturn1search1turn1search2

Рекомендований порядок:
1) `w:outlineLvl` абзацу → 2) `w:outlineLvl` стилю (через `basedOn`) → 3) назва/ID стилю → 4) Word COM OutlineLevel (fallback).

## 2) Як це реалізовано у GPT Super Табел
- XML‑парсер читає `document.xml` + `styles.xml`.
- Пріоритет: `outlineLvl` абзацу → outlineLvl стилю → назва/ID стилю.
- Якщо парсер не знаходить всі 1/2/3 — вмикається fallback через Word COM (`Paragraph.OutlineLevel`).

## 3) Аналіз поточного тестового файла (draft_all_in_one_fast_Статті.docx)
Симптом: у таблиці зібрались лише назви (Level 1), стилі для авторів/назв не застосовані.

Факт:
- У файлі фактично **немає рівнів 2/3** в OOXML. Бачимо переважно стилі `SECTION`, `heading 1` та різні `Normal`/`List Paragraph`.
- `heading 1` використаний і для назв статей, і для анотацій/ключових слів.
- Тому парсер отримує **тільки Level 1**, а інші рівні відсутні.
- Це не баг коду, а структура документа.

Статистика рівнів із парсера (XML):
- Level 1: 31
- Level 2: 1
- Level 3: 0

У такому документі потрібно **на рівні Word Outline** або руками виправляти стиль/outlineLvl для авторів і назв.

## 4) Що потрібно знайти у файлі “101 table test”
Для правильного автоматичного заповнення:
- Секції мають бути або `Heading 1`, або `outlineLvl=0`.
- Автори мають бути або `Heading 2`, або `outlineLvl=1`.
- Назви мають бути або `Heading 3`, або `outlineLvl=2`.

Якщо стиль один і той самий (напр. «Заголовок 2») використовується і для авторів, і для назв — тоді вирішальним має бути `outlineLvl`.

- окремо відмічу, де “Заголовок 2” має outlineLvl=3.
